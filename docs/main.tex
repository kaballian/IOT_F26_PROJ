\section{proposal}
The group would like to propose a mini-project that mixes elements from proprosal: project 1 \textit{Extended Node-RED data flow} and 
project 6 \textit{Web-of-things}.\\
In project 6, the project revolves around creating a RESTful IoT application with a sensor device, with continuous time-series data reports
and the use of Message oriented middleware to handle the data. In project 1, our main take away is support for sensor data, and data analytics 
on a smaller hardware target an \textit{Arduino} is proposed. It is assumed that any MCU will work in this case. 
The project will revolve around a device that both group members wants to create, which is a smart air filter \text{HEPA}, and an integrated IOT device
with controls \textit{actuates} fans to improve circulation, read data from an onboard temperature\_ humidity and air quality sensor, and then boardcasts
them in a IoT sense, like the weather station.\\
The focus is therefore \textbf{not} to be on any challenges that are outside of the scope of the course, ie. Power-conversion, specific sensor 
implementation or any related issues. But only those challenges that relates to the course itself. 

To keep it inline with a requirement from project 6: \textit{For the sensor devices, two variants exists, 
with different hardware configurations for the IOT device}, Two different, smaller, sensor systems will be constructed.
