\section{Project 6: Web-of-things ish}
This projects aim to combine the main topics of project 6 \textit{Web-of-things} with some
elements of project 1 \textit{Extended Node-RED dataflow} 
The project revolves around creation of a RESTful Iot application with a smart 
object, which provides continuous time-series data, in this case a smart air filter
incorporating air filters, for home use. The IoT device will in this case be an arduino 
esque MCU will which actuate hardware and take measurements and generally acts as the sensor.
This will be communicated to a Raspberry PI, which will implement a message-oriented middelware.\\

The rest of the project will then follow the pivot points of project 6,  such as:\\

\begin{itemize}
    \item time series data flow for subscribers
    \item a data model for IoT data
    \item Design and specification of a REST API front end
    \item A complete GIT repo
\end{itemize}


Hardware requirements:

\begin{itemize}
    \item 12V powersupply
    \item 5V linear regulator board for IOT device
    \item Raspberry PI
    \item \textit{Arduino} like breakout board 
    \item 12V PWM fans
    \item various other minor hardware
\end{itemize} 

The focus is therefore \textbf{not} to be on any challenges that are outside of the scope of the course, ie. Power-conversion, specific sensor 
implementation or any related issues. But only those challenges that relates to the course itself. 
Therefore any hardware implementation, that is not related or in spirit of the scope of project 6, is secondary to completion.

\subsection{Hardware System description}
Power is supply by a wall adapter and is set to 12V, to avoid any switch mode powersupplies to 
increase potential, as this voltage is required by PWM fans. The main controller of the whole system
is the Raspberry PI (RPi), which will host the messaging service and expose the API interface.
Communication between the \textit{sensor} and RPI is done with UART and communcation to all ancillary 
components (air quality sensor, climate sensor) is done with I2C.
